# Service API Bash client and random decision agent

[![DOI](https://zenodo.org/badge/1028413171.svg)](https://doi.org/10.5281/zenodo.18622181)


## What is in this repository?

This repository contains the OpenAPI-generated client for the service API in Bash. It is bundled with a script and a Dockerfile to make it take random decisions over a given configuration parameter in a certain range.

## What do I need to run this?

Bash, or Zsh, are necessary to install this into your system. Nonetheless, it is a better idea to simply use the Docker versions, for which you will need Docker and `docker-compose`.

## Repository structure

- `.openapi-generator`: `openapi-generator` metadata
- `docs`: Docs, autogenerated by `openapi-generator`
- `_serviceapi`: Script for Zsh autocompletion, autogenerated by `openapi-generator`
- `.openapi-generator-ignore`: Files that `openapi-generator` should ignore. Untouched since autogeneration
- `batch-openapi-generator.yaml`: Instructions used to generate the client for use with `openapi-generator-cli batch`.
- `docker-compose.yaml`: Docker Compose specification for the agent
- `Dockerfile`: Dockerfile for the agent, modified from the autogenerated version
- `Dockerfile-interactive`: Dockerfile to create a Docker image for interactive use with the API. It is the original version autogenerated by `openapi-generator`
- `serviceapi`: Client, autogenerated by `openapi-generator`
- `serviceapi.bash-completion`: Script for Bash autocompletion, autogenerated by `openapi-generator`
- `take_random_decisions.sh`: Shell script with the agent logic.

## Setup details

1. Set up the volumes and environment variables in Docker Compose to fit your environment.
2. `docker-compose up -d`.
3. The agent will run until stopped, reporting each value change in the results file, and waiting `$SLEEP_INTERVAL` between changes.

# Original OpenAPI generator README.md

## Overview

This is a Bash client script for accessing Service API service.

The script uses cURL underneath for making all REST calls.

## Usage

```shell
# Make sure the script has executable rights
$ chmod u+x serviceapi

# Print the list of operations available on the service
$ ./serviceapi -h

# Print the service description
$ ./serviceapi --about

# Print detailed information about specific operation
$ ./serviceapi <operationId> -h

# Make GET request
./serviceapi --host http://<hostname>:<port> --accept xml <operationId> <queryParam1>=<value1> <header_key1>:<header_value2>

# Make GET request using arbitrary curl options (must be passed before <operationId>) to an SSL service using username:password
serviceapi -k -sS --tlsv1.2 --host https://<hostname> -u <user>:<password> --accept xml <operationId> <queryParam1>=<value1> <header_key1>:<header_value2>

# Make POST request
$ echo '<body_content>' | serviceapi --host <hostname> --content-type json <operationId> -

# Make POST request with simple JSON content, e.g.:
# {
#   "key1": "value1",
#   "key2": "value2",
#   "key3": 23
# }
$ echo '<body_content>' | serviceapi --host <hostname> --content-type json <operationId> key1==value1 key2=value2 key3:=23 -

# Make POST request with form data
$ serviceapi --host <hostname> <operationId> key1:=value1 key2:=value2 key3:=23

# Preview the cURL command without actually executing it
$ serviceapi --host http://<hostname>:<port> --dry-run <operationid>

```

## Docker image

You can easily create a Docker image containing a preconfigured environment
for using the REST Bash client including working autocompletion and short
welcome message with basic instructions, using the generated Dockerfile:

```shell
docker build -t my-rest-client .
docker run -it my-rest-client
```

By default you will be logged into a Zsh environment which has much more
advanced auto completion, but you can switch to Bash, where basic autocompletion
is also available.

## Shell completion

### Bash

The generated bash-completion script can be either directly loaded to the current Bash session using:

```shell
source serviceapi.bash-completion
```

Alternatively, the script can be copied to the `/etc/bash-completion.d` (or on OSX with Homebrew to `/usr/local/etc/bash-completion.d`):

```shell
sudo cp serviceapi.bash-completion /etc/bash-completion.d/serviceapi
```

#### OS X

On OSX you might need to install bash-completion using Homebrew:

```shell
brew install bash-completion
```

and add the following to the `~/.bashrc`:

```shell
if [ -f $(brew --prefix)/etc/bash_completion ]; then
  . $(brew --prefix)/etc/bash_completion
fi
```

### Zsh

In Zsh, the generated `_serviceapi` Zsh completion file must be copied to one of the folders under `$FPATH` variable.
